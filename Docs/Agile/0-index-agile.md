# Agile Index - Factory-SaaS

## Proposito
Centralizar los artefactos Scrum para planificar, ejecutar y cerrar sprints con trazabilidad.

## Documentos
- `1-product-backlog.md`: Backlog unico priorizado.
- `2-sprint-backlog.md`: Trabajo seleccionado para el sprint activo.
- `3-definition-of-ready-dor.md`: Criterios de entrada.
- `4-definition-of-done-dod.md`: Criterios de salida.
- `5-sprint-planning-template.md`: Estructura de Sprint Planning.
- `6-daily-sync-log.md`: Registro diario de avance/bloqueos.
- `7-sprint-review-template.md`: Evidencias y validacion del incremento.
- `8-sprint-retrospective-template.md`: Mejora continua.

## Regla operativa
Todo item de backlog debe tener Anchor Doc en `Docs/` y respetar fronteras de independencia por app.
